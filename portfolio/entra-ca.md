# **Entra ID：条件付きアクセス（MFA 要求）**

## 1. 目的  
MFA を必須とする条件付きアクセスポリシーを作成し、  
ユーザーへの適用と有効化までを理解する。

## 2. 設計
- ユーザー  
  - Test.User-01  
- 対象アプリ  
  - Microsoft Entra 管理センター  
- アクセス制御  
  - 多要素認証（MFA）を要求  
- ポリシー状態  
  - 有効化（オン）

## 3. 手順

### 3-1. 条件付きアクセス画面を開く
1. **左メニュー**から  
   **「保護」 → 「条件付きアクセス」** をクリック。  
2. 上部タブで **「ポリシー」** が選択されていることを確認。  
3. この一覧画面をスクリーンショット：

<img src="../images/ca-policy-list.png" width="600">

### 3-2. 新しいポリシーを作成し、ユーザーを割り当てる

1. ポリシー一覧画面で **「新しいポリシー」** をクリック。  
2. **ポリシー名**に  
   `CA-MFA-Require-TestUser01` など分かる名前を入力。  
3. 左側のメニューから **「割り当て」 → 「ユーザーとグループ」** をクリック。  
4. **「ユーザーとグループを選択」** をクリック。  
5. 検索欄に `Test.User-01` を入力し、該当ユーザーにチェック。  
6. **「選択」 → 「完了」** をクリック。  
7. Test.User-01 が割り当てられている画面をスクリーンショット：

<img src="../images/ca-assignment-users.png" width="600">

### 3-3. アクセス制御で MFA を要求する（ca-access-control-mfa.png）
1. 左側メニューで **「アクセス制御」 → 「許可」** をクリック。  
2. 「アクセスを許可」にチェックが入っていることを確認。  
3. その下のオプションから **「多要素認証を要求」** にチェックを入れる。  
4. 右下の **「選択」** をクリックして確定。  
5. 「許可」画面で **MFA が要求される設定**になっている状態をスクリーンショット：

<img src="../images/ca-access-control-mfa.png" width="600">

### 3-4. ポリシーを有効化する（ca-policy-enabled.png）
1. 左側メニューで **「有効化」** をクリック。  
2. オプションから **「オン」** を選択。  
3. 画面右下の **「作成」 または 「保存」** をクリックしてポリシーを確定。  
4. ポリシー一覧に戻り、今作成したポリシーが  
   - 一覧に表示されている  
   - 状態が **「オン」** になっている  
   ことを確認。  
5. この一覧画面をスクリーンショット：  

<img src="../images/ca-policy-enabled.png" width="600">

## 4. 結果  
- **どこから条件付きアクセスに入るか**（保護 → 条件付きアクセス → ポリシー）  
- **誰に適用しているか**（Test.User-01）  
- **何を要求しているか**（MFA）  
- **ポリシーが有効かどうか**（オン）
